Reflection

How can I ensure that my code, program, or software is functional and secure?

I ensure my software is functional by writing unit tests that verify every requirement before deployment. During this course, I used JUnit to test valid inputs, invalid inputs, null values, duplicate IDs, boundary conditions, and exception handling. Running my tests with code coverage allowed me to verify that most of my code was exercised. I also incorporated input validation throughout my classes to prevent invalid data from entering the application, improving both reliability and security.

How do I interpret user needs and incorporate them into a program?

I begin by carefully reviewing the software requirements and identifying the expected behavior of each feature. I then design the application so that each requirement is implemented individually and verified through testing. Throughout this course, every requirement for the Contact, Task, and Appointment services was translated into both application logic and corresponding JUnit tests. This approach ensured that the final software matched the customer’s expectations.

How do I approach designing software?

I approach software design by first understanding the requirements before writing any code. I break larger problems into smaller classes and methods, each responsible for a specific task. After implementing the functionality, I immediately create unit tests to verify that each component behaves correctly. This process allows defects to be identified early, reduces technical debt, and produces software that is easier to maintain and expand in the future.
